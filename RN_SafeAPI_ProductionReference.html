💎ES6 Patterns & Default Values

1️⃣ Nullish Coalescing (??) vs OR (||)

यह MOST IMPORTANT concept है!

❌ WRONG (using ||)

const price = data?.price || 100; // अगर price = 0 है तो 100 बन जाएगा! ❌ const quantity = form?.qty || 1; // अगर qty = 0 है तो 1 बन जाएगा! ❌ const isDark = settings?.dark || false; // अगर dark = false है तो false ही रहेगा (coincidence!) ❌

✅ CORRECT (using ??)

const price = data?.price ?? 100; // अगर price = 0 है तो 0 ही रहेगा! ✓ const quantity = form?.qty ?? 1; // अगर qty = 0 है तो 0 ही रहेगा! ✓ const isDark = settings?.dark ?? false; // अगर dark = false है तो false ही रहेगा! ✓
Rule of Thumb:
?? (nullish) use करो जब 0, "", false valid values हों
|| (logical OR) use करो सिर्फ rare cases में

🔄Error & Data Normalization Checks

Normalization क्यों जरूरी है?
API कभी भी inconsistent data दे सकती है — null, undefined, wrong type, missing fields। बिना normalization के UI crash हो जाती है। तीन layers पर normalize करो: API Layer → State Layer → UI/Render Layer

🗂️ तीन Layers का Architecture

// LAYER 1: API Response आया → Normalize करो // LAYER 2: State में set करो → Validate करो // LAYER 3: UI में render करो → Safe guards लगाओ // Flow diagram: API Response ↓ normalizeApiResponse() Clean Data Object ↓ setStateWithDefaults() Safe State ↓ safeRender() Crash-Free UI ✅

1️⃣ Layer 1: API Response Normalization

API response को सीधे state में मत डालो — पहले normalize करो।

❌ WRONG — Raw API data directly use करना

const data = await response.json(); // Direct use — DANGEROUS! ❌ setUser(data.user); setProducts(data.products); // अगर data.user null है? // अगर data.products string है? // अगर field name बदल गई API में? // → CRASH!

✅ CORRECT — Normalize करके use करना

const raw = await response.json(); // Normalize first ✅ const clean = normalizeApiResponse(raw); setUser(clean.user); setProducts(clean.products); // सब defaults set हैं // Types verified हैं // Missing fields handled हैं ✓

Complete normalizeApiResponse() Function

const normalizeApiResponse = (raw = {}) => { // Step 1: Check if raw itself is valid if (!raw || typeof raw !== 'object' || Array.isArray(raw)) { console.warn('[Normalize] Invalid API response shape', raw); return getDefaultResponseShape(); } // Step 2: Normalize each field with type checks return { // String fields id: normalizeString(raw.id ?? raw._id ?? raw.userId), name: normalizeString(raw.name ?? raw.fullName ?? raw.display_name), email: normalizeString(raw.email ?? raw.emailAddress), // Number fields (API कभी string दे देती है "123") age: normalizeNumber(raw.age), price: normalizeNumber(raw.price ?? raw.amount), count: normalizeNumber(raw.count ?? raw.total), // Boolean fields (API "true"/"false" string दे सकती है) isActive: normalizeBoolean(raw.isActive ?? raw.active ?? raw.enabled), isVerified: normalizeBoolean(raw.isVerified ?? raw.verified), // Array fields items: normalizeArray(raw.items ?? raw.products ?? raw.data), tags: normalizeArray(raw.tags), // Nested object fields address: normalizeAddress(raw.address ?? raw.location), profile: normalizeProfile(raw.profile ?? raw.userProfile), // Date fields createdAt: normalizeDate(raw.createdAt ?? raw.created_at), updatedAt: normalizeDate(raw.updatedAt ?? raw.updated_at), }; }; // Default shape जब response invalid हो const getDefaultResponseShape = () => ({ id: null, name: '', email: '', age: null, price: 0, count: 0, isActive: false, isVerified: false, items: [], tags: [], address: null, profile: null, createdAt: null, updatedAt: null, });

Type-Specific Normalizer Helpers

// ─── String Normalizer ─────────────────────────────── const normalizeString = (val, fallback = '') => { if (val === null || val === undefined) return fallback; if (typeof val === 'string') return val.trim(); if (typeof val === 'number') return String(val); return fallback; }; // ─── Number Normalizer ─────────────────────────────── const normalizeNumber = (val, fallback = 0) => { if (val === null || val === undefined) return fallback; const num = Number(val); // Handles "123" → 123 return isNaN(num) ? fallback : num; }; // ─── Boolean Normalizer ────────────────────────────── const normalizeBoolean = (val, fallback = false) => { if (typeof val === 'boolean') return val; if (val === 'true' || val === 1 || val === '1') return true; if (val === 'false' || val === 0 || val === '0') return false; return fallback; }; // ─── Array Normalizer ──────────────────────────────── const normalizeArray = (val, fallback = []) => { if (!val) return fallback; if (Array.isArray(val)) return val.filter(item => item !== null && item !== undefined); if (typeof val === 'object') return Object.values(val); // {0: x, 1: y} → [x, y] return fallback; }; // ─── Date Normalizer ───────────────────────────────── const normalizeDate = (val) => { if (!val) return null; const date = new Date(val); return isNaN(date.getTime()) ? null : date.toISOString(); }; // ─── Nested Object Normalizer ──────────────────────── const normalizeAddress = (addr) => { if (!addr || typeof addr !== 'object') return null; return { street: normalizeString(addr.street ?? addr.line1), city: normalizeString(addr.city), state: normalizeString(addr.state ?? addr.province), country: normalizeString(addr.country, 'IN'), pincode: normalizeString(addr.pincode ?? addr.zip ?? addr.postal_code), }; };

💥UI Crash Protection Guards

React Native में UI crash क्यों होती है?
ये crashes production में सबसे ज्यादा होती हैं और users को blank screen दिखती है। नीचे सब common crash scenarios और उनके fixes हैं।

🔴 Crash Type 1: Null/Undefined Property Access

❌ CRASH Code

✅ PROTECTED Code

🔴 Crash Type 3: Non-String in <Text> Component

React Native में <Text> के अंदर सिर्फ String या Number होना चाहिए — Object, Array, Boolean CRASH करते हैं!

❌ CRASH Scenarios

✅ SAFE Rendering
